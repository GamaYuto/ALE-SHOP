## Pasos para correr el proyecto localmente

1. Abrir una terminal en la carpeta del proyecto ALE-SHOP

2. Instalar dependencias
- Ejecutar:
  - `npm install`

3. Iniciar el servidor de desarrollo
- Ejecutar:
  - `npm run dev`

4. Abrir la app en el navegador
- Vite muestra una URL local, normalmente:
  - `http://localhost:5173`

5. Si quieres previsualizar la versión de producción
- Primero build:
  - `npm run build`
- Luego preview:
  - `npm run preview`

### Notas importantes
- El proyecto usa `vite` como servidor de desarrollo.
- El archivo principal es index.html; también hay una entrada adicional admin.html configurada en vite.config.js.
- Asegúrate de tener Node.js y npm instalados antes de ejecutar los comandos.

## Manual para subir los cambios a Firebase

### 1. Preparar el entorno local
1. Abre una terminal en la carpeta del proyecto ALE-SHOP
2. Instala las dependencias del proyecto:
   - `npm install`

### 2. Instalar Firebase CLI (si no está instalado)
- Puedes instalar globalmente:
  - `npm install -g firebase-tools`
- O usar con `npx` sin instalar globalmente:
  - `npx firebase-tools --version`

### 3. Login en Firebase
- Inicia sesión en tu cuenta de Firebase:
  - `firebase login`
- Si usas `npx`:
  - `npx firebase login`

### 4. Verificar el proyecto Firebase configurado
El archivo .firebaserc ya indica el proyecto:
- Alias: `ALESHOP`
- Proyecto: `ale-shop-7b472`

Para ver el proyecto activo:
- `firebase projects:list`
- O con `npx`:
  - `npx firebase projects:list`

Si deseas usar explícitamente el alias:
- `firebase use ALESHOP`
- o `npx firebase use ALESHOP`

### 5. Generar el build de producción
El despliegue usa la carpeta `dist` definida en firebase.json.

Ejecuta:
- `npm run build`

Verifica que exista la carpeta `dist/` tras el build.

### 6. Desplegar a Firebase Hosting
Ejecuta:
- `firebase deploy --only hosting`
- o con `npx`:
  - `npx firebase deploy --only hosting`

### 7. Verificar el sitio publicado
- Al finalizar, Firebase CLI mostrará la URL de hosting.
- Abre esa URL en el navegador para confirmar que los cambios se aplicaron.

---

## Notas importantes

- firebase.json ya tiene configurado `public: "dist"` y reglas de rewrites para que todas las rutas apunten a index.html.
- Si cambias el proyecto de Firebase o el alias, actualiza .firebaserc.
- Si necesitas desplegar también Firestore u otros servicios, usa `firebase deploy` sin `--only hosting`, pero para este proyecto normalmente basta con hosting.

Este es el flujo típico para subir cambios a Firebase desde este proyecto.
